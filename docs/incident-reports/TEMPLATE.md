## Submission Guidelines

When submitting an incident report, please:

1. **Be Factual**: Stick to verifiable facts, not speculation
2. **Cite Sources**: Provide links to primary sources (blog posts, news articles, official statements)
3. **Technical Detail**: Include technical analysis when possible
4. **No Sensationalism**: Use clear, professional language
5. **Privacy**: Redact personal information unless already public
6. **Updates**: Note if information is still emerging or unconfirmed

### Severity Levels

- **CRITICAL**: Data loss, system compromise, security breach affecting many users
- **HIGH**: Significant malfunction, production impact, security vulnerability
- **MEDIUM**: Moderate impact, functionality issues, limited user impact
- **LOW**: Minor issues, edge cases, no significant harm

### Status Definitions

- **Confirmed**: Incident verified by multiple sources or official acknowledgment
- **Alleged**: Incident reported but not independently verified
- **Under Investigation**: Incident being investigated, details still emerging

### Verification Levels

- **High**: Multiple independent sources, official statements, technical analysis available
- **Medium**: Single credible source, some technical details, or platform acknowledgment
- **Low**: Single source, limited details, or conflicting reports
